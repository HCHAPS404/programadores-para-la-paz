Guardar reportes en una lista dentro del servidor es un buen ejercicio para entender cómo funciona una API sencilla.
Primero envío un POST y el servidor agrega el dato; luego con GET /reportes veo todo lo guardado.
En el ejemplo los datos viven solo en memoria, así que si apago el servidor se pierden.
Aun así, la lógica es la misma que usaría una app real antes de conectar una base de datos.
Si varios vecinos reportan lo mismo, tener la lista ayuda a ver patrones.
Consultar los reportes después de registrarlos permite hacer seguimiento sin depender de mensajes sueltos.
Para una comunidad, eso puede mejorar la organización de problemas del entorno.
